    # WSDL Discovery without a Live Endpoint

    Visual Studio provides a useful service discovery and proxy generation tool for quickly building WCF clients. Unfortunately, Disney were unable to provide access to the web service endpoint which was behind their WAN firewall.

    The initial development used the WSDL to create a dummy test service on our development server to mimic the Disney service. Although the Service Contract and Data Contracts were duplicated exactly, the lower level services differed and the client provided did not work at the Disney site.

    # WSCF.blue

    ThinkTecture, a German consulting firm, provide a Visual Studio extension called WSCF.blue which is able to fill the gaps in the Microsoft provided tools. Their add-on can read a WSDL file and produce both client and server proxies with the required configuration. Ref: http://wscfblue.codeplex.com/

    WSCF.blue is a Visual Studio .NET 2008/2010 add-in that provides the following features:

    • A WSDL Wizard that allows the developer to step through the creation of a WSDL from one or more XSDs.

    • A Data Contract Generator (similar to XSD.exe, XSDObjectGen.exe and Svcutil.exe) that generates the . NET equivalent of the XSD types.

    • A Service/Endpoint Stub (SVC) Generator and

    • A Client Proxy Generator.

    • A Generate Data Contract Code feature that supports the selection of multiple XSD/WSDL source files. (More Information)

    • Support for C# and VB.NET code generation.

    • You can choose if operation methods on your service class will throw a NotImplementedException, call an implementation method in a partial class, or will be defined as abstract methods. (More Information)

    • Force the SOAP actions (Action and ReplyAction) applied to each operation contract follow the standard WCF format: <namespace>/<service>/<operation>[Response]

    • A Paste XML as Schema option that generates a schema for a block of XML in the clipboard.

    • Errors found in your WSDL are reported in a WSCF.blue pane in the Output window.

    For the endpoints and the client proxy, WSCF also generates the necessary configuration file.
